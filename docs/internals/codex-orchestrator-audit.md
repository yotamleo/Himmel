# Codex-as-orchestrator readiness audit (HIMMEL-739 draft)

Draft status: timeboxed, evidence-based breadth pass. This audits capabilities the
himmel parent/orchestrator loop assumes from the Claude Code runtime, then names
what a Codex/Hermes orchestrator has today and whether to port, substitute, or
accept the gap.

Definitions:

- `PORTED` means repo evidence shows a Codex/Hermes analog already exists and is
  wired or documented as live enough for this draft.
- `PORTABLE` means the capability is not native today, but repo evidence shows a
  clear script/hook/skill substrate that can carry it with bounded work.
- `CLAUDE-ONLY` means the capability depends on Claude Code runtime semantics,
  Claude status/cache surfaces, or Claude's built-in tools enough that the honest
  recommendation is to accept the gap or design a new vehicle.
- `UNVERIFIED` means the files read here did not prove the claim. It is not a
  negative proof.

## Suggested port order

1. Close proof gaps before adding features: codex-direct external-write e2e,
   Codex secret-read e2e, and lifecycle `additionalContext` live probes.
2. Decide the semantic-classifier stance: deterministic guard table only, or a
   new auxiliary judgment layer with explicit hard-veto cases.
3. Define a Codex/Hermes resume vehicle before touching scheduler permissions;
   the current sanctioned path launches `claude`, not Codex or Hermes.
4. Define quota/cap telemetry for Codex/Hermes before porting auto-arm; without
   a trustworthy reset signal, a self-rearm hook would be false confidence.
5. Promote worker dispatch to an explicit lane registry: dispatcher command,
   worktree behavior, model selector, artifact root, guard proof, liveness probe.
6. Keep Codex skills thin wrappers around shared scripts; avoid duplicating the
   Claude command logic into Codex-only prose.
7. Treat panel-only `pr-check` as a deliberate degraded mode until a Codex-native
   reviewer/adjudicator is specified.
8. Keep generated `AGENTS.md` as the portable rule projection; do not rely on
   Claude project memory for Codex/Hermes decisions.
9. Add startup health checks to any Codex lane dispatch so truncated prompts or
   ignored hooks fail visibly before work is trusted.
10. Update `docs/internals/lane-parity.md` after each proof lands; this audit is
    a draft snapshot, not the living conformance source.
